Getting Started with Garp

Learn how to create your first Garp site and get up and running quickly.

Search Documentation

Installation

Garp is distributed as a single binary with no dependencies. Download the latest version for your platform: 

# macOS (Homebrew)
brew install garp

# Linux/macOS (curl)
curl -L https://github.com/example/garp/releases/latest/download/garp-$(uname -s)-$(uname -m) -o garp
chmod +x garp
sudo mv garp /usr/local/bin/

# Windows (PowerShell)
Invoke-WebRequest -Uri "https://github.com/example/garp/releases/latest/download/garp-windows-amd64.exe" -OutFile "garp.exe"

Creating Your First Site

Initialize a new Garp project with the garp init command: 

# Create a new site
garp init my-awesome-site
cd my-awesome-site

# Start the development server
garp serve

# Build for production
garp build

This creates a new directory with all the necessary files and folder structure for your Garp site.

Project Structure

A typical Garp project has the following structure: 

my-awesome-site/
├── site/                    # Built HTML files (auto-generated)
├── content/                 # Your markdown content
│   ├── index.md            # Homepage content
│   └── posts/              # Blog posts or pages
├── templates/              # HTML templates
│   ├── _template.html      # Base template
│   └── post.html          # Post template
├── input.css              # Tailwind CSS input file
├── tailwind.config.js     # Tailwind configuration
└── bin/                   # Build scripts
    ├── build-css          # CSS compilation
    └── build-search-index # Search indexing

Writing Content

Garp uses Markdown files with YAML frontmatter for content. Here's an example: 

---
title: "My First Post"
date: "2024-01-15"
author: "Jane Doe"
tags: ["getting-started", "tutorial"]
description: "Learn how to create your first post with Garp"
---

# Welcome to My First Post

This is the content of my first post written in **Markdown**.

## Features

- Easy to write

- Automatic HTML generation
- Full-text search integration

Development Workflow

1. Write Content

Create Markdown files in the content/ directory with YAML frontmatter.

2. Customize Styling

Edit input.css and tailwind.config.js to customize your site's appearance.

3. Preview Changes

Use garp serve to preview your site locally with live reloading.

4. Build & Deploy

Run garp build to generate static files for deployment.
