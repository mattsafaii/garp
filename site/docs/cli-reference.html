CLI Reference

Complete reference for all Garp CLI commands and options.

Search Commands

garp init

Initialize a new Garp project with scaffolding and configuration files.

garp init [project-name] [flags]

Arguments

  • project-name - Name of the project directory to create

Flags

  • --force - Overwrite existing directory if it exists
  • --template - Specify a template to use (default: basic)
  • --no-git - Skip Git repository initialization

Examples

# Create a new project
garp init my-blog

# Force overwrite existing directory
garp init my-blog --force

# Use a specific template
garp init my-blog --template=portfolio

garp serve

Start the development server with live reloading and automatic building.

garp serve [flags]

Flags

  • --port, -p - Port to serve on (default: 3000)
  • --host - Host to bind to (default: localhost)
  • --open, -o - Open browser automatically
  • --verbose, -v - Enable verbose logging
  • --no-reload - Disable live reloading

Examples

# Start server on default port
garp serve

# Start server on custom port
garp serve --port 8080

# Start server and open browser
garp serve --open

# Start with verbose logging
garp serve --verbose

garp build

Build the site for production, generating optimized static files.

garp build [flags]

Flags

  • --css-only - Build only CSS assets
  • --search-only - Build only search index
  • --watch, -w - Watch for changes and rebuild
  • --verbose, -v - Enable verbose output
  • --clean - Clean build artifacts before building

Examples

# Build everything
garp build

# Build with verbose output
garp build --verbose

# Build only CSS
garp build --css-only

# Build and watch for changes
garp build --watch

# Clean and build
garp build --clean

garp deploy

Deploy your site to various hosting platforms.

garp deploy [platform] [flags]

Platforms

  • netlify - Deploy to Netlify
  • vercel - Deploy to Vercel
  • s3 - Deploy to AWS S3
  • github-pages - Deploy to GitHub Pages

Flags

  • --config - Path to deployment configuration file
  • --dry-run - Show what would be deployed without actually deploying
  • --force - Force deployment even if no changes detected

Examples

# Deploy to Netlify
garp deploy netlify

# Dry run deployment
garp deploy netlify --dry-run

# Deploy with custom config
garp deploy s3 --config deploy.yaml

Global Flags

These flags are available for all commands:

  • --help, -h Show help for command
  • --version Show version information
  • --config Path to configuration file
  • --quiet, -q Suppress non-error output

Configuration

Garp can be configured using a garp.config.json file in your project root: 

{
  "site": {
    "title": "My Awesome Site",
    "description": "A site built with Garp",
    "baseURL": "https://example.com"
  },
  "build": {
    "outputDir": "site",
    "assetsDir": "assets"
  },
  "server": {
    "port": 3000,
    "host": "localhost",
    "liveReload": true
  },
  "search": {
    "enabled": true,
    "indexContent": true,
    "excludePaths": ["admin/**"]
  }
}

Environment Variables

These environment variables can be used to configure Garp:

  • GARP_CONFIG Path to configuration file
  • GARP_PORT Default port for serve command
  • GARP_VERBOSE Enable verbose logging (true/false)
