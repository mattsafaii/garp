Why Another Static Site Generator?

The static site generator landscape is crowded with excellent tools like Jekyll, Hugo, Gatsby, and Next.js. So why build another one? The answer lies in our specific requirements and philosophy:

  • ✓ Zero Configuration: Works out of the box with sensible defaults
  • ✓ Single Binary: No runtime dependencies or complex installation
  • ✓ Built-in Search: Full-text search without external services
  • ✓ Modern CSS: Tailwind CSS integration with automatic optimization
  • ✓ Fast Builds: Parallel processing and efficient file handling

We wanted a tool that could generate a complete, production-ready website from Markdown files with absolutely no configuration required, while still being extensible for advanced use cases.

Technical Architecture

Garp is built with Go and leverages several key technologies to achieve its goals:

Core Components

Markdown Processing

Uses goldmark for CommonMark-compliant Markdown parsing with extensions for syntax highlighting, tables, and task lists.

Template Engine

Go's built-in html/template package provides secure, fast template rendering with automatic XSS protection.

CSS Processing

Tailwind CSS v4+ integration with automatic purging and optimization for minimal output file sizes.

Search Integration

Pagefind provides client-side full-text search with fuzzy matching and zero server requirements.

Build Pipeline

1. Parse frontmatter and content from Markdown files
2. Process templates with parsed data
3. Generate HTML files with proper linking
4. Compile Tailwind CSS with used classes only
5. Build search index from generated content
6. Optimize and minify assets
7. Generate sitemap and RSS feeds

Performance Optimizations

Performance was a key consideration throughout development. Here are some of the optimizations we implemented:

Build Performance

  • → Parallel Processing: Content processing and asset compilation happen concurrently
  • → Incremental Builds: Only modified files are reprocessed during development
  • → Template Caching: Parsed templates are cached in memory for reuse
  • → File Watching: Efficient file system monitoring with debouncing

Runtime Performance

  • → CSS Purging: Unused Tailwind classes are automatically removed
  • → Asset Optimization: Images and other assets are optimized for web delivery
  • → Client-Side Search: No server requests needed for search functionality
  • → Minimal JavaScript: Only essential JS is included in the output

Benchmark Results

On a typical blog with 100 posts, Garp can complete a full build in under 500ms, including CSS compilation and search index generation. Incremental builds during development typically complete in under 50ms.

Getting Started

Ready to try Garp for your next project? Getting started is simple: 

# Install Garp
curl -L https://github.com/example/garp/releases/latest/download/garp-$(uname -s)-$(uname -m) -o garp
chmod +x garp && sudo mv garp /usr/local/bin/

# Create a new site
garp init my-blog
cd my-blog

# Start developing
garp serve

That's it! You'll have a fully functional static site with search, optimized CSS, and a development server with live reloading.
